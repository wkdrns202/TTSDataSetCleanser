### 1.5 현재 최적 파라미터

```
SEG_SEARCH_WINDOW = 25       AUDIO_PAD_MS = 100
SKIP_PENALTY = 0.01          PREATTACK_SILENCE_MS = 400
MATCH_THRESHOLD = 0.50       TAIL_SILENCE_MS = 730
CONSEC_FAIL_LIMIT = 10       ONSET_SAFETY_MS = 30
MAX_MERGE = 5                OFFSET_SAFETY_MS = 80
Whisper: medium (Tier 1) + large (Tier 2, 실패 항목만)
```

### 1.6 핵심 발견 사항

- **temperature=0 금지**: 한국어 연음/경음화/음운변동으로 deterministic decoding은 치명적
- **전방 탐색만 사용**: 후방 탐색은 한국어 조사/어미 유사 패턴으로 중복 매칭 유발
- **탐색 범위 25줄**: 100~500줄 범위는 false-match cascading 야기
- **Tier 2 복구**: large 모델이 medium 실패의 ~15%를 복구하나, 8GB VRAM에서 순차 로딩 필요
- **한국어 micro-pause 문제**: -었습니다 등 형식 어미의 미세 정지를 Whisper가 세그먼트 경계로 오인

## 2. SAM Data Engine과의 구조적 비교

### 2.1 SAM Data Engine 핵심 (Kirillov et al., Meta AI)

SAM의 Data Engine은 3단계로 구성되며, 핵심은 **모델이 데이터를 생성하고, 그 데이터로 모델이 개선되는 선순환 구조**이다.

```
[SAM]  annotate → train model → better model → better annotations → ...
[현재]  optimize parameters around fixed Whisper → plateau at Whisper's ceiling
```

| Stage | SAM | 본 TTS 파이프라인 |
|-------|-----|-------------------|
| 1. Assisted-Manual | 어노테이터 + SAM 인터랙티브; 6회 재훈련 | 파라미터 반복 최적화 (7회, 35%→95%) |
| 2. Semi-Automatic | confident mask 자동 + 인간 보완; 5회 추가 재훈련 | Tier 1/2 계층 검증 (유사하나 정적) |
| 3. Fully Automatic | 1.1B 마스크 자율 생성, stability/confidence 기반 필터링 | **미달성** — 고정 Whisper에 의존 |

### 2.2 핵심 차이: 선순환 구조의 부재

현재 파이프라인은 **Whisper를 블랙박스로 사용**하며, 파이프라인의 출력이 Whisper 자체를 개선하지 않는다. 이것이 SAM과의 근본적 차이이다. SAM은 모델이 데이터를 생산하고, 그 데이터가 다시 모델을 개선하는 폐루프(closed-loop)를 형성한다.

### 2.3 SAM의 99.1%에 대한 재해석

SAM 논문의 99.1%는 **최초 모델의 세그먼트 적중률이 아니라 최종 데이터셋 구성비**이다. 즉:

- 모델이 세그먼트한 결과 중 **높은 신뢰도(stability check 통과)만을 데이터셋에 편입**
- 나머지는 버리거나 추가 처리를 거침
- 결과적으로 데이터셋의 품질을 보장하는 것은 모델의 raw 적중률이 아니라 **선별 전략**

**시사점**: 본 파이프라인에서도 데이터셋의 다양한 측면을 다각도로 검증하고, 가장 합리적으로 추정되는 부분만 높은 점수를 부여하여 데이터셋에 편입하는 **선별적 데이터 구성(Selective Data Composition)** 전략을 도입해야 한다.

## 3. SAM 기법의 음성 정렬 적용 아이디어

### 3.1 Stability Check → 정렬 신뢰도 스코어링

동일 세그먼트에 대해 **복수의 디코딩 설정**(beam size, temperature 샘플)으로 Whisper를 실행하여:
- 전사 결과 수렴 → 고신뢰 정렬 (자동 수락)
- 전사 결과 분산 → 리뷰 대상 또는 2차 처리

### 3.2 Multi-Hypothesis Matching → 모호성 인식 정렬

세그먼트가 복수의 대본 라인에 매칭 가능할 때:
- **Top-K 정렬 후보**를 점수와 함께 생성
- 가설 그래프(hypothesis graph)를 유지하여 전역적으로 해소
- 한국어 조사/어미 false-positive 문제에 직접 대응

### 3.3 Selective Data Composition → 신뢰 기반 데이터셋 구성

SAM의 데이터셋 구성 전략에 착안:
- 정렬 결과를 **다중 관점으로 검토** (Whisper 전사, 음향 특성, 텍스트 길이 비율, 세그먼트 경계 등)
- 각 관점에서 일관되게 높은 점수를 받는 세그먼트만 데이터셋에 편입
- 불확실한 세그먼트는 즉시 탈락이 아닌 **대기열(pending pool)**에 배치하여 추후 개선된 모델로 재처리

## 4. 향후 연구 방향 (A/B/C/D)

### (A) Data Engine — 파이프라인/모델 레벨 개선

#### A-1: 파이프라인 레벨 Data Engine

**목표**: 현재의 Whisper 기반 음성정렬 파이프라인에 Data Engine 개념을 적용하여 파이프라인 자체를 개선한다.

- **현 파이프라인의 일반화 검증**: 다른 녹음 데이터셋(다른 화자, 다른 도메인)에 적용하여 정렬 정확도가 유지되는지 확인
- 현재 95%가 이 특정 데이터셋에만 유효한지, 일반적인 한국어 TTS 데이터에 보편적으로 적용 가능한지 검증
- Stability check, multi-hypothesis matching 등 SAM 기법의 음성 정렬 적용
- **선별적 데이터 구성** 전략 도입: 다각도 검증 후 고신뢰 세그먼트만 편입

#### A-2: ASR 모델 레벨 Data Engine (인코더-디코더)

**목표**: STT 모델의 인코더-디코더 레벨로 내려가, Data Engine이 ASR 모델 자체를 개선하는 방향을 탐색한다.

- 검증된 speech-text 쌍을 활용한 Whisper 파인튜닝
- 파인튜닝된 Whisper로 파이프라인 재실행 → 실패 복구 → 새로운 훈련 데이터 확보 → 반복
- SAM의 3단계 자동화 경로를 ASR 도메인에 매핑
- **리스크**: 단일 화자/도메인 과적합 → A-1의 일반화 검증이 선행 조건

```
[A-2 선순환 구조]
검증된 4,000쌍 → Whisper fine-tune → 개선된 Whisper
→ 파이프라인 재실행 → 5% 실패 일부 복구
→ 새 훈련 데이터 → Whisper 재학습 → ...
```

### (B) 모델 학습: F5TTS 및 평가 체계

- **F5TTS Phase 1/2 완성**: 정제된 데이터셋으로 TTS 모델 학습
- **MOS(Mean Opinion Score) 기반 평가 체계 구축**: 자동 메트릭이 포착하지 못하는 지각적(perceptual) 품질 측정
- **테스트 데이터 확보**: 학습/평가 분리를 위한 held-out set 구성
- TTS 모델의 출력 품질이 곧 데이터 파이프라인의 실질적 검증 수단

### (C) 추론/배포

- 기존 도메인(koreanAIvoice.com)에 배포된 모델의 **교체/추가/업데이트**
- 실제 사용자 피드백 확보
- 아키텍처적 우선순위는 낮으나 실사용 피드백 가치 높음

### (D) TTS↔ASR 피드백 루프 — SAM식 선순환 구현

**목표**: TTS 합성 음성을 ASR로 역검증하여 데이터 품질을 개선하고, 합성 데이터로 모델을 발전시키는 선순환 구조를 구현한다.

```
[피드백 루프]
TTS가 음성 합성 → ASR이 역전사 → 입력 텍스트와 비교
→ 불일치 시: TTS 발음 오류 OR ASR 인식 오류
→ 이 신호로 양쪽 모두 개선 → 반복
```

**SAM 데이터셋 구성비 전략 적용**:
- SAM의 99.1%가 데이터셋 구성비인 점에 착안
- 데이터셋의 여러 부분을 **다양한 관점에서 검토** (음향 품질, 전사 정확도, 프로소디 일관성, 합성 역검증 등)
- 가장 합리적으로 추정되는 부분만 높은 점수를 부여하여 데이터셋에 편입
- 불확실한 데이터는 탈락이 아닌 **재처리 대기열**로 관리
- 모델이 개선될수록 대기열에서 복구 가능한 데이터가 증가하는 선순환

이 구조가 A-2와 결합되면 **SAM의 fully automatic stage에 해당하는 자기지도(self-supervised) 검증 루프**가 완성된다.
